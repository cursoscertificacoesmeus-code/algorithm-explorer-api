Atualização da API para o Frontend - Conectividade do Grafo (Explicação Revisada)

Olá!

Esta é uma atualização sobre a explicação do campo `isConnected` na resposta do endpoint de processamento de grafos, para maior clareza.

---
### Parte 2: Processamento de Grafo
---

#### Resposta da API (Response Body) - ATUALIZAÇÃO RECENTE

O campo `isConnected` na resposta do endpoint `POST /api/graph/process` agora tem uma explicação mais precisa:

- **`isConnected` (boolean):** Indica se o grafo é conectado.
  - Para **todos os tipos de grafo** (não direcionados, direcionados e mistos), esta verificação agora se refere à **conectividade fraca**.
  - Ou seja, `true` se todos os nós são alcançáveis a partir de qualquer outro nó, **ignorando a direção das arestas**. Se o grafo subjacente (onde todas as arestas são tratadas como não direcionadas) é conexo, então `isConnected` será `true`.

**Exemplo de Resposta de Sucesso (200 OK) com o campo `isConnected`:**
```json
{
  "nodeCount": 3,
  "edgeCount": 2,
  "weighted": false,
  "graphType": "Directed",
  "message": "Graph processed successfully. It is a directed graph with 3 nodes and 2 edges.",
  "nodeDetails": [
    {
      "id": "A",
      "inDegree": 0,
      "outDegree": 1
    },
    {
      "id": "B",
      "inDegree": 1,
      "outDegree": 1
    },
    {
      "id": "C",
      "inDegree": 1,
      "outDegree": 0
    }
  ],
  "isConnected": true
}
```

Com o campo `isConnected` agora refletindo a conectividade fraca para todos os tipos de grafo, o frontend pode ter uma compreensão mais consistente sobre a conectividade geral do grafo.
